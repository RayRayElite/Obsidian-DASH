# How This System Works

Obsidian DASH works best when each note type has a narrow job instead of trying to be an all-purpose container.

## Core Principle

Keep action, reference, review, personal context, and AI guidance separate enough that each stays easy to scan.

## Operational Layers

### Master Task Hub

Use the Master Task Hub as the cross-project action surface.

It is for:

- active tasks
- project status and focus metadata
- repeating work
- queue management across projects
- archived completion history

It is not for:

- large reference dumps
- long-form project thinking
- personal context notes
- AI behavior rules

### Project Notes

Use project notes for the deeper context behind one project.

They are for:

- goals
- risks
- constraints
- decisions
- supporting links
- review history

They should explain the project, not replace the Master Task Hub.

### Daily Logs

Daily logs are the human-readable record of what actually happened.

They are for:

- tracked activity
- work completed
- consumables, symptoms, and sleep context
- reflections
- follow-through history

They are not the place to maintain future plans.

### Weekly Reviews And Reports

Weekly reviews and period reports summarize patterns, drift, and progress.

They are for:

- review by exception
- noticing blockers
- seeing trend changes
- deciding what to change next

### Basic Information

Basic Information is durable personal context that changes slowly.

It is for:

- stable personal facts
- persistent constraints
- long-lived preferences
- useful background context for AI

### AI Guardrails

AI Guardrails is not a biography note. It is a behavior-rules note.

It is for:

- tone preferences
- planning preferences
- risk tolerance
- what the AI should avoid
- how to interpret weak evidence

### Current Season

Current Season holds temporary context.

It is for:

- current month priorities
- temporary obligations
- short-lived constraints
- recent life context that matters right now

### Decision Journal

Decision Journal stores why important decisions were made.

It is for:

- a decision
- the reason
- expected effect
- when to revisit it

## Healthy Information Flow

Use this flow:

1. Capture action in the Master Task Hub.
2. Keep deeper context in project notes.
3. Let the dashboard generate daily and periodic artifacts.
4. Move stable personal context into Basic Information.
5. Move AI behavior instructions into AI Guardrails.
6. Move temporary life context into Current Season.
7. Preserve important choices in the Decision Journal.

## Failure Modes To Avoid

- putting project reference material directly into the Master Task Hub
- storing durable personal context in daily logs
- mixing AI behavior rules into random notes
- treating weekly reviews as the place where action lists live permanently
- allowing project notes to become unstructured archives

## Good Default Rule

If something needs to be acted on soon, it belongs in the Master Task Hub.

If something explains work, it belongs in a project note.

If something records what happened, it belongs in logs or reviews.

If something is stable personal context, it belongs in Basic Information.

If something tells AI how to behave, it belongs in AI Guardrails.
