# Document Freshness Checklist

Use this checklist during weekly review, monthly review, or system maintenance.

## Evergreen Notes To Check

- [ ] Basic Information still reflects reality.
- [ ] AI Guardrails still matches the desired AI behavior.
- [ ] Current Season still reflects the current month or phase.
- [ ] System Map still matches the real note and folder structure.
- [ ] Master Task Hub project metadata is still populated for active projects.
- [ ] Project notes still have current focus, risks, and constraints.
- [ ] Repeating task rules still match reality.
- [ ] Cleanup and review notes are not being used as permanent storage.

## Signs A Note Is Stale

- it references projects that no longer matter
- it contains old constraints that are no longer true
- it has headings with empty placeholder content for too long
- the dashboard depends on it but the note no longer reflects real behavior
- AI outputs keep misunderstanding the same context

## Review Questions

- What note is creating the most confusion right now?
- What note is carrying too many roles at once?
- What context should be stable but is still trapped in daily logs?
- What context should be temporary but is still living in evergreen notes?
- What project note has become a reference dump instead of a working note?

## Outcome Rule

Every freshness review should end with one of these:

- update the note
- split the note
- archive part of the note
- remove dead content
