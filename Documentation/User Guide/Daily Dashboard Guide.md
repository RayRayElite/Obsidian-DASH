# Daily Dashboard Guide

The daily dashboard is the operational surface of DASH. It should help you decide what to do, track what matters, and review what changed without becoming a second vault inside your vault.

## What The Dashboard Is For

- seeing today clearly
- capturing and promoting work quickly
- tracking sessions, habits, calendar events, and state data
- surfacing reminders, friction, and drift early

## Core Daily Workflow

### Start of day

- Open the dashboard.
- Check notifications, calendar pressure, and suggested focus.
- Pull the next task from the hub or Kanban instead of inventing new work.

### During the day

- Use quick add when work belongs in a project.
- Track sessions only when they improve planning or review.
- Keep dashboard notes short and operational.

### End of day

- close or park open loops
- end the logical day cleanly
- let DASH write the day into readable history

## Cards That Matter First

- Action Queue or your main action surfaces
- Session Deck
- Weekly Agenda
- Timeline Search
- Notifications and hero utilities

Everything else can come later if it proves useful in actual use.

## UI Customization

The dashboard is meant to be rearranged. You do not have to accept the first layout as permanent.

### Reorder Cards By Drag And Drop

- drag a card by its header to move it earlier or later in the dashboard
- pinned cards stay prioritized near the top, so pin only the few cards you want to stay important
- hidden cards are not deleted; they are only removed from the current view until you show them again

### Use The Layout Editor For Deeper Changes

Open the layout editor from the sliders icon in the hero footer or with `Alt+Shift+L`.

Use it when you want to:

- reorder cards more deliberately
- pin cards that should stay near the top
- hide cards you do not want on screen
- change card width between default, half width, and full width
- reset the dashboard back to the default layout

### View Modes Matter

The dashboard has three view modes, and they change how much fits on screen.

- mobile: tighter vertical reading and narrower stacking
- compact: the default balanced mode for normal desktop use
- widescreen: better when you want more cards visible at once on a large display

Cycle view modes from the hero footer or with `Alt+Shift+V`.

## Hero Header And Utility Controls

The hero area is not decorative only. It is the dashboard control rail.

### Main Hero Buttons

- `New project`: create a project and its related note or hub structure
- `Add to project`: quick-capture a task straight into a project and active section without opening the hub first
- `Review mode`: open the structured project review workflow
- `Repair day`: fix logical-day issues when tracking drifted or something was left open incorrectly
- `Notifications`: open reminders, warnings, and system notices that need attention

### Hero Footer Utility Icons

- view mode icon: switch between mobile, compact, and widescreen layouts
- book icon: open the DASH documentation center
- keyboard icon: show dashboard shortcut help
- sliders icon: customize dashboard layout
- notebook icon: generate the weekly review note
- warning triangle icon: generate the recurring friction patterns note
- bar chart icon: generate the weekly report
- line chart icon: generate the monthly report
- trophy icon: generate the gamification report
- refresh icon: sync repeating project tasks

### Support Footer Ribbon

- at the bottom of the dashboard page, DASH shows a separate support ribbon instead of taking space from the hero area
- use the `Support on Ko-fi` button if you want to support development; it does not affect dashboard behavior or unlock features

### Status Pills In The Hero Footer

The hero status row gives fast context without opening deeper cards.

- date: the active dashboard date
- archived: how many tasks have already been completed and archived today
- stale: how many projects currently look stale
- mood and energy: the latest state snapshot
- hidden: how many cards are currently hidden from the layout

## Notifications And Undo

- use the notifications button when the badge count is non-zero or the bell looks alert-colored
- notifications can surface calendar reminders, system setup issues, task pressure, and logical-day warnings
- some notifications let you jump straight into the right repair or setup action
- destructive dashboard actions also create a temporary undo banner so you can reverse the last removal without rebuilding it manually

## Session Deck Customization

Session Deck can also be customized.

- reorder trackers when the default button order does not match your workflow
- rename trackers to fit your own language
- hide trackers you do not use often
- change tracker colors when you want clearer visual separation

The goal is to make the visible controls reflect your real routine instead of forcing you into the default vocabulary forever.

## Layout Tips

- Hide cards you do not use often.
- Pin only a few cards.
- Use compact, widescreen, and mobile modes intentionally instead of leaving the view in whatever state happened last.

## Logical Day Rule

DASH uses a logical day instead of assuming that your real day resets at midnight.

- use `Begin logical day` when you wake up and are actually starting the day
- use `End logical day` when you are going to bed and the day is really over
- sleep tracking, daily logs, and reports depend on those two actions being used consistently

If you stay up past midnight, DASH will keep treating that time as part of the same day until you end it. That is what keeps late-night work and sleep timing attached to the day you actually mean.

## Where History Lives

- daily logs for factual recall
- weekly and monthly reports for interpretation
- timeline search for retrieval across sessions, tasks, events, and text logs
