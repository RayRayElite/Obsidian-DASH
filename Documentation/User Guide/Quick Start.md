# Quick Start

Use this guide if you want DASH useful quickly without turning setup into its own project.

## 1. Finish Core Setup

- Run the first-run setup wizard.
- Set the `Master Task Hub` path.
- Confirm your daily log, weekly report, and monthly report folders.
- Leave AI off until the manual workflow already makes sense.

## 2. Make Sure The Hub Is Readable

Your `Master Task Hub` should work as plain markdown before you care about advanced automation.

Recommended baseline per project:

- a `## Project Name` heading
- useful metadata such as `Status::` or `Project Note::`
- active work sections such as `### Now`, `### Next`, `### Later`, `### Waiting`, and `### Reference`

## 3. Learn The Three Main Surfaces

- Daily Dashboard: daily action, tracking, reminders, and quick review.
- Master Task Hub: readable source of truth for project work.
- DASH Kanban: faster visual board on top of the same data.

## 4. Use A Narrow First-Week Workflow

- Start the day from the dashboard with `Begin logical day` when you wake up.
- Pull or quick-add work from projects.
- Track only the data that actually helps planning.
- End the logical day with `End logical day` when you go to bed so sleep tracking and day tracking stay trustworthy.

### Logical Day Rule

DASH uses a logical day, not a strict midnight-to-midnight day.

- your tracked day starts when you use `Begin logical day`
- your tracked day ends when you use `End logical day`
- sleep timing, day logs, and reports depend on using those two actions consistently

This matters most if you are awake late at night or after midnight. Until you end the day, DASH still treats that time as part of the same day.

## 5. Add Features In This Order

1. Dashboard and Master Task Hub
2. Project notes
3. DASH Kanban
4. Weekly and monthly reports
5. AI Workspace
6. Compiled research wiki

## Common First Mistakes

- Over-customizing before the defaults have had time to prove useful.
- Turning every note into both an action list and a reference note.
- Enabling AI before the source notes are readable.
- Treating Kanban as a separate system instead of a faster view.
