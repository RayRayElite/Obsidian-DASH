# Archive Policy

This system stays usable when old material leaves the active surfaces on purpose instead of by accident.

## Archive Active Tasks

Completed checklist items should leave active project sections and move into `Completed Archive`.

That keeps:

- current sections actionable
- project history preserved
- daily logs aligned with what was actually completed

## Archive Reference Material

Reference material should be moved out of the Master Task Hub once it stops helping short-term action decisions.

Move it into:

- the project note
- a supporting note
- a reference folder note if the material grows large

## Archive Project State

A project should usually move out of active posture when:

- it has no current next step
- it is intentionally paused
- it is waiting on an external dependency for a long time
- it is no longer worth active review frequency

Use `Incubating` or `Someday` instead of pretending it is still active.

## Archive Review Material

Old weekly reviews, cleanup notes, and project review notes should stay searchable but should not become the main place you work from.

They are for history and pattern review, not live action management.

## Archive Rule Of Thumb

If something is no longer helping today’s decisions but might help future understanding, archive it instead of deleting it.
