# Support

Obsidian DASH is being prepared for public community beta release.

## Where To Get Help

- Bug reports: GitHub Issues
- Feature requests: GitHub Issues
- Beta feedback and first-impression reports: GitHub Issues using the beta feedback template
- Community discussion or announcements: Obsidian forum showcase post or Discord, when available

## Current Support Channel Decision

For the public beta, GitHub Issues are the primary support and feedback channel.

- use GitHub Issues for bugs, feature requests, and beta feedback
- use forum posts or Discord for visibility or announcements, not as the primary support queue

## What To Include In A Bug Report

- what you expected to happen
- what actually happened
- exact steps to reproduce it
- screenshots if the issue is visual
- whether AI, Kanban, or budgeting was enabled
- whether the issue happens in a fresh vault or only in an existing vault
- any relevant console or notice output

## Support Boundaries

- DASH is actively maintained, but still in beta.
- Issues are welcome.
- Feature requests may be accepted selectively.
- The plugin is optimized first for practical desktop use, not for every possible workflow.
- Planned ideas are not promises until they are explicitly shipped.

## Known Limitations

- The current beta should be treated as desktop-only.
- Budgeting and gamification are still experimental.
- Advanced Trello-style Kanban extras such as comments, assignee images, non-image attachments, and percentage progress bars are not part of the current release promise.
- AI workflows are optional and depend on separate model setup.

## What Counts As A Beta Blocker

- the plugin fails to load or the main dashboard does not open
- DASH Kanban breaks core project workflow
- the release assets or install instructions are wrong enough to block a new user
- recovery commands or docs are missing for common broken states

## What Feedback Is Still Valuable But Usually Not Blocking

- requests for future mobile support
- requests for richer Trello-like Kanban depth
- requests for deeper nutrition tracking
- polish suggestions that do not break the core dashboard, hub, or Kanban workflow

## Future Ideas Policy

Possible later ideas should not be treated as promises until they are actively being built and documented.

- this especially applies to mobile sync and voice-triggered workflows
- beta support responses should frame those as possible later directions, not committed roadmap items

## Good First Recovery Steps

If something looks wrong, try the smallest relevant recovery action first:

- refresh generated artifacts
- run the relevant repair command
- rebuild the AI index if retrieval seems stale
- reload Kanban templates and themes if custom assets changed
- confirm settings paths still point to the intended notes and folders
