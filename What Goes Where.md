# What Goes Where

Use this as the fast reference for where information should live.

## Master Task Hub

Put these here:

- active checklist items
- project metadata
- short queue items
- repeating tasks
- archived completions

Do not put these here:

- long reference notes
- design rationale longer than a few lines
- personal background context
- AI behavior rules

## Project Notes

Put these here:

- project summary
- why it matters
- definition of done
- risks
- constraints
- decisions
- supporting links
- project-specific review history

## Daily Logs

Put these here:

- day history
- tracked work and sessions
- health and body context
- daily reflections
- what actually got completed

## Weekly Reviews And Reports

Put these here:

- patterns
- exceptions
- drift
- accomplishments
- follow-up review notes

## Basic Information

Put these here:

- stable personal facts
- enduring preferences
- long-lived health context
- persistent constraints

## AI Guardrails

Put these here:

- desired AI tone
- planning behavior preferences
- what AI should avoid
- how cautious or assertive AI should be

## Current Season

Put these here:

- current priorities
- temporary obligations
- seasonal constraints
- short-lived life context

## Decision Journal

Put these here:

- meaningful decisions
- rationale
- expected outcome
- revisit date or trigger

## Quick Rule

Action goes in the hub.

Explanation goes in project notes.

History goes in logs and reviews.

Stable personal context goes in Basic Information.

AI behavior rules go in AI Guardrails.
