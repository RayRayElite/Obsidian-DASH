# Contributing

Thanks for your interest in Obsidian DASH.

This project started as a personal daily-use tool and is now being prepared for a public community beta. That means contributions are welcome, but the project still has a strong product direction and not every suggestion will be accepted.

## Before You Open A Pull Request

- Open an issue first for larger changes.
- Keep fixes focused and narrow.
- Do not bundle unrelated cleanup with a feature or bug fix.
- Keep the Master Task Hub as the readable source of truth for project work.
- Avoid adding background complexity when a clearer explicit workflow is enough.

## Best Contribution Targets

- bug fixes
- documentation improvements
- usability polish
- recovery and repair flow clarity
- public-beta release hygiene

## Less Likely To Be Accepted Quickly

- major workflow rewrites
- broad architectural refactors without a user-facing problem
- features that increase complexity before beta stability improves
- mobile-sync assumptions that push the desktop-first product direction off course

## Development Notes

- install dependencies with `npm install`
- build with `npm run build`
- deploy locally with `npm run build:deploy -- "D:/Game Dev/Projects/.obsidian/plugins/daily-dashboard"`

## Pull Request Guidance

- explain the user-facing problem being solved
- explain why the chosen change is the smallest reliable fix
- mention any commands used for validation
- include screenshots if the change is visual
